# 静态资源分享网站页面原型图

## 1. 文档说明

本文档使用 Markdown + ASCII 线框图描述页面原型，用于在正式视觉设计和编码前，先确认信息层级、页面结构和主要交互。

原型重点围绕以下目标：

- 首页突出“独立开发软件”
- 全站支持资源目录扩展
- 搜索入口足够明显
- 资源详情页能清晰展示百度网盘下载信息
- 支持白天 / 黑夜主题切换
- 页面预留充足的二次元图片展示区域
- 整体动画轻盈、精美，不走后台面板风

## 2. 设计方向

### 2.1 Visual Thesis

网站整体气质应更像“二次元作品橱窗 + 资源目录”，而不是常见的博客流或后台面板。

### 2.2 Content Plan

- Hero：品牌、站点定位、全站搜索、主视觉插画位
- Support：独立开发软件专区、角色图/软件封面组合陈列
- Detail：资源目录、分类、标签、公告、主题图占位区
- Final CTA：引导进入资源详情和下载

### 2.3 Interaction Thesis

- 首页 Hero 搜索作为主入口
- 列表页通过筛选栏快速缩小结果范围
- 详情页通过下载面板和复制提取码操作提升转化
- 主题切换时，背景色、发光和插画容器做柔和过渡
- 图片位支持悬浮、视差、渐入等轻动画

### 2.4 Theme & UI Strategy

- 主题切换基于 CSS 变量，不在组件内写死颜色
- 交互组件优先使用成熟组件库，例如 `Radix UI + shadcn/ui`
- 组件库只负责交互骨架，视觉皮肤统一定制为偏二次元风格
- 图片区域默认留空，使用“插画位 / 立绘位 / 横幅位 / 截图位”作为占位命名

### 2.5 Motion Direction

- 首屏：标题、按钮、搜索框、主视觉分段进入
- 背景：光点、花瓣、星屑或色块低频漂浮
- 卡片：hover 时上浮、发光、边框渐亮
- 详情页：下载面板和角色图有轻微视差与淡入

### 4.3 移动端布局说明

- 顶部导航折叠为菜单按钮
- Hero 搜索框占满宽度
- 主视觉插画位改为置顶横幅或单图
- 软件作品改为纵向卡片流
- 分类入口变成可横向滚动标签栏
- 主题切换保留在顶部栏右侧
- 插画位在移动端不删除，只压缩为单列模块

## 14. 主题、组件与动画说明

- 主题切换必须可见且全站一致，建议放在 Header 右侧
- 所有颜色、阴影、背景发光必须来自 CSS 变量
- 推荐使用成熟组件库提供以下控件：`Switch`、`Drawer`、`Tabs`、`Dialog`、`Carousel`
- 视觉上不直接使用组件库默认皮肤，而是统一包裹二次元风格主题
- 动画控制以“柔和、轻盈、层次感”为主，不做大幅度晃动

## 15. 移动端通用适配原则

- 顶部导航改为抽屉菜单或折叠菜单
- 搜索框在首页和搜索页均保持首屏可见
- 资源列表卡片改为单列
- 详情页下载面板优先级高于长正文
- 标签和分类筛选可横向滑动
- 插画位在移动端保留，但优先纵向排列，避免强行双列压缩

## 16. 图片占位区命名建议

- `主视觉插画位`：首页 Hero 最大图片位
- `角色立绘位`：右侧人物或看板娘图片位
- `专题横幅位`：分类页、公告页顶部横幅
- `软件封面位`：资源卡片封面图位置
- `截图位`：详情页软件界面图
- `氛围图位`：用于补足二次元氛围的场景图或横向视觉图

## 17. 页面设计结论

本套原型的核心设计取向是：

- 首页做“作品入口 + 搜索入口 + 主视觉舞台”
- 资源页做“目录检索 + 专题氛围入口”
- 详情页做“信息说明 + 下载转化 + 角色/截图展示”
- 公告和关于页做“信任补充 + 品牌氛围补充”

这样可以让用户先快速认识你，再快速找到资源，最后顺畅完成百度网盘下载。
